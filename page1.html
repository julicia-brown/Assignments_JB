Things we've learned so far:

  1. Semantic Tags
    1. Semantic Tags are HTML tags that are used not for function but for logical structures.
    2. Semantic Tags are often used to define the parts of a webpae and often used to assign them their distinct looks.
    3. Without CSS these tags will not affect the look of your page. They greatly help with organizing your code too!
  2. Cascading Stylesheets (CSS)
    1. Cascading stylesheets also known as CSS is used to define what out HTML should look like using a set of rules. These rules work similar to if/then statements we're used to in everyday life.
    2. There are three ways to implement CSS into your HTML. In line which goes into the line of each code. Embedded which goes into a style tag on your HTML coding page and lastly as a separate CSS file which then is linked.
    3. The reason why it is called casacading is becaus eof how browsers will read this style sheets. They read in a trickle down cascading method where the inline style or default style will always take precedents if what is in the CSS sheet is not applicable.
  3. CSS Box Model
    1. Many if not all HTMLM elements can be thought of as boxes. This is how many browsers think of them and function.
    2. There are different parts of your box ,odel that you can control using css for it's design and structure.
    3. With this box you can design where on your page an element sits and what that element looks like among the others.
